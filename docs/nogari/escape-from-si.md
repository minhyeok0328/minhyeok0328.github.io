구글 서치콘솔? 이랑 워드프레스 세팅하는데 도움을 많이 받고 나서 생애 첫 개인 블로그를 열었었습니다.

그 때 당시에 디자인 패턴 공부중이였기 때문에 php로 직접 MVC 패턴을 만들어서 깃에 올리고
관련 내용을 블로그에도 작성하며 굉장히 재밌게 블로그를 운영하고 있었는데

```
<?php
	Class Model{
		
		protected $db;
		protected $param;
		protected $sql;

		function __construct(){//db연결
			$this->param = Application::getParam();
			$this->db = new PDO("mysql:host=localhost;dbname=mvc;charset=utf8;","root","");
			$this->db->setAttribute(PDO::ATTR_DEFAULT_FETCH_MODE,PDO::FETCH_OBJ);//mysql 정보를 object형식으로
		}

		//쿼리문 설정
		protected function query($sql = false){
			if ($sql) $this->sql = $sql;//$this->sql로 초기화
			$res = $this->db->query($this->sql);//$res에 쿼리정보 저장
			if (!$res) {//$res가 없을 경우 sql문,오류메세지 출력
				echo $this->sql;
				echo "<pre>";
				print_r($this->db->errorInfo());
				echo "</pre>";
			} else{
				return $res;//쿼리값 반환
			}
		}

		protected function fetch($sql = false){
			if ($sql) $this->sql = $sql;
			return $this->query()->fetch();
		}

		protected function fetchAll($sql = false){
			if ($sql) $this->sql = $sql;
			return $this->query()->fetchAll();
		}

		protected function cnt($sql = false){
			if ($sql) $this->sql = $sql;
			return $this->query()->rowCount();
		}

		//배열을 컬럼 형태로 반환
		protected function get_column($arr,$cancel){
			$cancel = explode("/", $cancel);
			$column = "";
			foreach ($arr as $key => $val) {
				if (!in_array($key, $cancel)) {
					$column .= ", {$key}='{$val}'";
				}
			}
			return substr($column, 2);
		}

		//action의 value값에 따라 쿼리문 변경
		protected function get_query($column){
			switch ($this->action) {
				case 'insert':
					$sql = "INSERT INTO {$this->table} SET ";
				break;
				case 'update':
					$sql = "UPDATE {$this->table} SET ";
				break;
				case 'delete':
					$sql = "DELETE FROM {$this->table} ";
				break;
			}
			$sql .= $column;
			return $this->query($sql);
		}

	}
```

## 끔찍.

> 최근들어 SI 쪽도 많이 개선되었다고 들었는데 제 회사는 예외였나봅니당.

야근 한다길래 그래도 적당히 하겠거니 했는데 무슨 며칠 씩 집에 못가고 면도도 못하고 씻지도 못한 채
창 밖에서 해가 뜨는 광경을 몇 번을 보고나서야 집에 보내주는 끔찍한 야근이 많았습니다.

주말이고 공휴일이고 회사로 가는 일이 잦았고 프로젝트 일정도 빡빡한 데다가 한 번에 2~4개의 프로젝트에 다리를 걸치고 있어서
출근 시간은 명확하나 퇴근시간을 알 수 없고 연차를 쓰려고 해도 싹 다 반려되는 경우가 다반사였으며 심한 년도는
연차 1년 15개 중 2~3개만 쓰는 끔찍한 일도 벌어졌습니다. (심지어 휴가 쓴 사유들도 야근하다 몸살나서, 병무청에 신검 받으러 등등)

야근하다 새벽에 차 끊겨서 택시타고 집 간거 지결서 제출했는데 이사님이 저희 팀장님을 까며 왜 회사돈을 맘 대로 쓰냐 갈구고
프로젝트는 견적서 보니까 14명 투입된다고 적혀있는데 실 작업자는 4~6명인 경우가 다반사에 야근했던 말건 주말이건
사정 안 봐주고 회사 블로그를 강제로 쓰게 하는 등 나열하면 끝도 없습니다. (이 때 블로그가 사라졌습니다.)

퇴사하기엔 이미 병역특례 TO를 받아서 복무중이였고, 정말 이직하고 싶었으나 주말에도 회사에 계속 나가니 블로그 쓸 시간도 없고, 개인 공부 할 시간도 없었거니와
이직도 병역특례 회사로만 가능했기 때문에 탈출이 불가능 해 보였습니다.

프로젝트 끝나면 가끔 중간에 텀이 생기긴 했는데 번아웃이라는걸 몸소 느껴보며 코드를 쳐다도 보지 않았다고 합니다 하하.
오래 쉬지도 못했긴 하지만.

## 그래도 공부는 하고 싶다

제가 재직할 당시 저희 회사의 대부분의 프로젝트들이 쇼핑몰 솔루션을 구매하여 고객사에 요청에 맞게 커스텀을 하는 식으로 프로젝트들을 많이 진행했습니다.
DB는 MySQL, 백엔드는 PHP 7.2 ~ 7.4? 쯤 사용했으며 Codeigniter 라는 프레임워크를 사용했었습니다.

프론트 쪽에 관심이 많았던 터라 쇼핑몰 프로젝트 할 떄는 조금 아쉽긴 했으나
다행히 백오피스 시스템이나 신규 구축 하는 프로젝트가 있어
나름 이것저것 써볼 수 있는 기회들이 있었기 때문에 당시 SPA의 개념이나 ES6 이상 문법을 잘 모르시고
jQuery 위주로 사용했었던 팀원 분들 한테 vue로 프로젝트를 해보면 어떻겠냐고 권유를 했습니다.

react랑 vue 중에서 굉장히 고민을 했습니다만, 그 때 당시에는 react hook이 없었고 class를 사용했었기 때문에
vue에 비해서는 상대적으로 러닝커브가 높았고, 프로젝트를 슬슬 시작했어야 해서 팀원분들이 따로 공부할 시간이 크게
없었기 때문에 상대적으로 직관적이고 쉬운 vue로 결정을 하였습니다.

과정 자체는 사실 순탄치는 않았습니다. 제 사수분도 REST API를 처음 만들어 보셨기 때문에
백엔드와 json으로 데이터를 주고받는 과정에서 꽤나 많은 커뮤니케이션을 주고받았었고
token을 주고받을 때 프론트 어디에 저장하는지, 어떤식으로 발급프로세스를 만들지 등의 내용들이 오고 갔습니다.

초반에는 제게 프론트 관련해서 알려주실 분이 없었기 때문에 삽질도 많이 하고 작업이 힘들었긴 했지만 이렇게 고생했기에
이 시기에 꽤나 많이 성장했지 않나 싶습니다.

XSS, CSRF, SQLI 등과 같은 보안 관련한 지식도 공부하게 되었고, 중간에 자바할 줄 아시는 분 들이 전부 퇴사하면서
자바 스프링 맛도 살짝 보고(찍먹 수준) 시간 날 때 마다 퇴근을 미뤄가며 잘 모르던 시절에 다 import해서 쓴 코드 리펙토링 해가며
트리쉐이킹 해가면서 번들 사이즈도 3~4mb씩 줄이면서 router에 Lazy Loading도 적용해서 로딩시간도 0.5s 넘게 줄이고

화면에 표시되는 데이터가 많아 느려지는게 체감이 많이 돼서 Virtual Scroll도 붙여보고
Mixin으로 재사용되는 부분들 최대한 줄여나가며 vue 3 나오기 전에 composition api 베타로 공개했길래 그걸로
코드 리펙토링도 하면서 틈틈히 프로젝트들을 개선해 나갔습니다.

## 퇴사

정말 회사업무가 바쁘긴 했지만 그렇다고 다른 회사로 이직을 시도해 보지 않은 건 아니였습니다.

학교 선배님이 우리회사 한 번 와볼래? 라고 하셔가지고 새벽 3시까지 야근하고 아침 8시까지 과제 풀어가면서 제출했었던 기억도 있습니다만,
면접 날짜 잡히고 얼마 지나지 않아 연락이 오더니 **전문연구요원만 되서 산업기능요원은 아쉽게 지원할 수 없다** 라고 연락이 왔었습니다ㅠㅠ
진짜 살면서 이렇게 간절하게 열심히 해 본게 손에 꼽는데 참 아쉬웠죠.

그 후로 원티드에 이력서 올려놓고 여기저기 지원도 했고, 뭔가 입사제의? 같은것도 왔었는데 메일로 산업기능요원
되는지 물어보면 다들 안된다고 하시고 또 중간에 훈련소도 갔었기 때문에 타이밍이 안좋아서 결국 복무를 여기서 마치게 되었습니다.

사실 기존 팀원들이 다 퇴사하고 나서 회사에서 저에게 굉장히 잘해주시기는 했고, 옆에 개발팀으로 병합되면서 프론트 쪽으로 도움
많이 주시는 사수분이랑 같이 프로젝트 진행도 하면서 상황이 그래도 점점 나아지고는 있는게 체감이 되고 있었습니다.

경기 침체 떄문에 취업시장이 별로 안좋다고 하기도 하고 드디어 사수분께 배우는 것도 생기기 시작했기 때문에 이번년도 까지는
다녀볼까 라는 생각까지도 했었는데 프로젝트 진행하면서 사수분 께서 저한테 하셨던 말이 있었는데 구조 잘 짜고 코드 퀄리티 개선하는건 좋은데
일정이 너무 촉박하니 일단 빨리 하는게 좋겠다 라고 하셨었거든요

근데 보통 프로젝트를 빨리 끝낸 다음에 유지보수 기간 끝내고 빨리 다음 프로젝트를 받으려고 하니 리펙토링은 사실 제가 야근을
사서 하거나 하지 않으면 할 기회가 없었기 떄문에 언제 다시 취업할 지는 모르겠지만 그 동안 쌓아뒀던 공부들을 마저 하며 시간을 보내는게
좋겠다고 판단이 들어 바로 퇴사를 하게 되었습니다.
