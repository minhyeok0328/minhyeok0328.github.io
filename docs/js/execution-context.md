# 실행 컨텍스트란?

## 실행 컨텍스트 (Execution Context)
Javascript 코드를 실행할 수 있는 환경을 뜻하며, 모든 Javascript코드들이 실행 컨텍스트 내부에서 동작합니다.
(eval 이라는 것도 있긴 하지만 현재는 거의 쓰지 않으니 해당 내용은 넘어가도록 하겠습니다.)

## 개요
우리가 사용하는 브라우저는 기본적으로 개발자 들이 작성하는 코드들을 이해하지 못합니다.
그래서 브라우저와 컴퓨터가 이해할 수 있는 형식으로 변환하는 과정이 필요합니다.

이 역할을 해주는 것이 브라우저에 있는 Javascript 엔진입니다.
각 브라우저를 만드는 기업에서 개발해서 사용하고 있는데 주로 많이 사용하시는 브라우저 들을 예로 들자면


- **Google Chrome**, Edge: V8
- **FireFox**: SpiderMonkey
- **Internet Explorer**~~(이미 죽었지만 예의상 적었습니다)~~: Chakra 


브라우저를 만드는 업체에서 각각 Javascript 엔진을 개발해서 사용하고 있습니다.
이렇게 개발된 Javascript 엔진들이 Javascript 코드를 변환 및 실행처리를 할 수 있는 환경을 만드는데 이것이 아래 설명드릴 실행 컨텍스트 입니다.

실행 컨텍스트에는 현재 실행중인 코드, 실행에 도움이 되는 모든 것이 포함되어 있으며, 런타임 동안 코드가 파서에 의해 구문이 분석되고, 변수 및 함수가 메모리에 저장되며 실행 가능한 바이트 코드가 생성되고 코드가 실행되는 순서입니다.

Javascript 에는 두 가지의 실행 컨텍스트 종류가 있습니다.

## 전역 컨텍스트(Global Execution Context)

Javascript 엔진은 스크립트 파일을 수신할 때 먼저 글로벌 실행 컨텍스트를 생성합니다.
글로벌 실행 컨텍스트는 함수 내부에 없는 모든 Javascript 코드가 실행되는 기본적인 실행 컨텍스트 입니다.

Javascript가 실행되는 환경에 따라 브라우저 에서는 Window, Node.js 에서는 global 객체가 됩니다.

## 함수 컨텍스트(Function Execution Context)
함수가 호출될 때 마다 Javascript엔진은 함수 실행 컨텍스트를 생성해서 해당 함수 내의 코드를 실행합니다.
